== Règles Absolues

=== Git

**INTERDICTION FORMELLE** de commit/push/merge sans permission explicite de l'utilisateur.

Commandes autorisées (lecture seule) : `git status`, `git diff`, `git log`, `git show`
Commandes interdites (sauf ordre) : `git add`, `git commit`, `git push`, `git merge`, `git rebase`

=== Tests

L'agent NE DOIT JAMAIS lancer de tests (`./gradlew test`, etc.) lors de la procédure de fin de session SANS permission explicite de l'utilisateur.

=== Sécurité

* Jamais de secrets en dur dans le code
* Jamais de credentials dans les fichiers de config
* Jamais de tokens API commités dans Git

=== Fichiers de configuration — RÈGLE ABSOLUE DE SÛRETÉ

**PRINCIPE** : Tout fichier de configuration (YAML, JSON, XML, TOML, properties, .env, etc.) peut contenir des données sensibles non versionnées. Modifier un tel fichier sans précaution met en danger l'intégrité du travail du développeur.

**INTERDICTIONS** :

* **NE JAMAIS écraser** un fichier de configuration existant avec un `Write` complet si on peut faire un `Edit` partiel
* **NE JAMAIS remplacer** des valeurs sensibles existantes (tokens, mots de passe, clés API, URLs de production) par des valeurs factices ou placeholders
* **NE JAMAIS modifier** un fichier sans avoir **d'abord** vérifié s'il est dans `.gitignore` (`git check-ignore <fichier>`) — un fichier ignoré n'a pas de `git restore` possible

**Procédure OBLIGATOIRE** avant modification d'un fichier de configuration :

. `git check-ignore <fichier>` : Vérifier si le fichier est versionné
. `git ls-files -- <fichier>` : Confirmer s'il est tracké
. Si le fichier **n'est pas tracké** (pas dans git) : **CRÉER UNE SAUVEGARDE** avant toute modification (`cp <fichier> <fichier>.bak`)
. Si le fichier contient des valeurs sensibles : **les PRÉSERVER intactes**, ne jamais les remplacer par des placeholders
. Privilégier **l'edit partiel** (modification ciblée) plutôt que **l'écriture complète** (écrasement total)
. Après modification : **vérifier** que les valeurs sensibles sont toujours présentes

=== Fichiers personnels — Interdiction de modifier

_L'agent NE DOIT JAMAIS_ modifier, écraser ou supprimer un fichier personnel de l'utilisateur (`~/.gitconfig`, `~/.bashrc`, `~/.ssh/config`, `~/.zshrc`, etc.) _SANS permission explicite de l'utilisateur_.

=== Méthodologie de Travail

* **1 fichier à la fois** : Modifier un seul fichier avant de passer au suivant
* **Validation immédiate** : Compiler après chaque modification
* **Contexte léger** : Maximum 50k tokens EAGER (~3000 lignes)
* **Règle d'or** : Mieux vaut 5 sessions de 20 minutes qu'une session de 2 heures avec debugging chaos. Si la conversation > 50 messages ou > 3 fichiers modifiés en parallèle, splitter en nouvelle session.

== Architecture du Projet

=== Structure

La gouvernance agent (`AGENT.adoc`, `PROMPT_REPRISE.adoc`, `.agents/`, `opencode.json`)
vit dans le répertoire plugin : `graphify-gradle/graphify-plugin/`. C'est le pattern standard
foundry (comme `plantuml-gradle/plantuml-plugin/`, `bakery-gradle/bakery-plugin/`).

[source,bash]
----

├── settings.gradle.kts              # NO include()
├── gradlew / gradle/
├── .gitignore
├── graphify-plugin/                 # ⭐ RACINE DE TRAVAIL + BUILD INDÉPENDANT
│   ├── AGENT.adoc                   # Directives complètes (ce fichier)
│   ├── PROMPT_REPRISE.adoc          # Mission session en cours (EAGER)
│   ├── opencode.json                # Config opencode
│   ├── build.gradle.kts             # Plugin build (java-gradle-plugin + maven-publish)
│   ├── settings.gradle.kts          # rootProject.name = "graphify-plugin"
│   ├── gradlew + gradle/            # Wrapper INDÉPENDANT
│   ├── .agents/                     # Gouvernance agent

│       ├── main/kotlin/com/cheroliv/graphify/

=== Composants Clés

* **GraphifyPlugin** : Point d'entrée du plugin — enregistre l'extension et la tâche
* **GraphifyExtension** : DSL — `rootDir`, `outputFile`, `excludePatterns`
* **ScanWorkspaceTask** : Cœur du plugin — walk récursif (Files.walkFileTree), extraction nœuds/arêtes/communautés, sérialisation Jackson
* **GraphModel** : Contrat JSON — GraphNode, GraphEdge, GraphCommunity
* **api-key-pool** (`../api-key-pool/`) : Pool de clés API avec quota tracking/réinitialisation/journalisation

== Session Procedure

=== Début de Session

1. Lire `PROMPT_REPRISE.adoc` pour la mission en cours
2. Lire `.agents/INDEX.adoc` pour les règles absolues
3. `git status` — vérifier l'état du dépôt
4. `./gradlew compileKotlin` — vérifier que le plugin compile
5. `../api-key-pool/gradlew compileKotlin -p ../api-key-pool` — vérifier que le lib compile

=== Fin de Session

**Mots-clés déclencheurs** : "nouvelle session", "je quitte", "session terminée", "à plus tard", "on arrête là"

**Procédure obligatoire en 6 étapes** :

1. **Créer l'archive LAZY** : `.agents/sessions/{N}-{type}-{sujet}.adoc`
2. **Mettre à jour PROMPT_REPRISE.adoc** — Mission N+1 (max 70 lignes)
3. **Mettre à jour SESSIONS_HISTORY.adoc** : `.agents/SESSIONS_HISTORY.adoc` — Ajouter la session terminée
4. **Mettre à jour INDEX.adoc** : `.agents/INDEX.adoc` — Date, session en cours N+1, roadmap, portefeuille
5. **Mettre à jour TEST_COVERAGE_ANALYSIS.adoc** (si tests modifiés)
6. **Mettre à jour COMPLETED_TASKS_ARCHIVE.adoc** : `.agents/archives/COMPLETED_TASKS_ARCHIVE_{mois}.adoc`

**Aucune étape ne peut être marquée OK si le fichier n'a pas été effectivement modifié et vérifié.**

== Anti-Patterns

[cols="1,3,3,3"]
|===
| # | Anti-pattern | Symptôme | Correction
| 1 | **Agent sans `opencode.json`** | L'agent démarre nu, pose des questions basiques déjà dans AGENT.adoc | `opencode.json` doit toujours contenir `"instructions": ["AGENT.adoc"]`
| 2 | **PROMPT_REPRISE fantôme** | PROMPT_REPRISE existe mais n'est pas mis à jour en fin de session | L'étape 2 de la procédure de fin de session est obligatoire
| 3 | **Archive vide** | Template avec placeholders "TODO" | Remplir intégralement à chaque session
| 4 | **Session trop longue** | > 50 messages ou > 3 fichiers parallèles | Splitter. 5x20min > 1x2h.
| 5 | **Split-brain .agents/** | .agents/ dupliqué dans racine et sous-projet | .agents/ est UNIQUE dans le sous-projet plugin (RACINE DE TRAVAIL). Jamais à la racine consumer.
| 6 | **Gouvernance à la racine consumer** | AGENT.adoc/.agents/ dans graphify-gradle/ au lieu de graphify-plugin/ | Toujours dans le sous-projet plugin. Pattern standard foundry : plantuml-gradle/plantuml-plugin/, bakery-gradle/bakery-plugin/, graphify-gradle/graphify-plugin/.
|===

== Bootstrap d'un Nouveau Projet — 6 étapes

Pour initialiser la gouvernance agent sur un projet vierge :

. **opencode.json** — 6 lignes, `{ "instructions": ["AGENT.adoc"] }`. Toujours en premier.
. **Dossiers** — `mkdir -p .agents/sessions/ .agents/archives/ .agents/encyclopedies/`
. **AGENT.adoc** — fichier maître (~200-250 lignes) : règles absolues, structure projet, conventions, cercles de confiance
. **PROMPT_REPRISE.adoc** — mission session 1 (max 70 lignes)
. **Fichiers LAZY structurants** : INDEX.adoc (EAGER), SESSIONS_HISTORY.adoc, TEST_COVERAGE_ANALYSIS.adoc
. **Portefeuille transverse** — ajouter le nouveau projet dans le tableau Portefeuille de TOUS les INDEX.adoc des autres projets ET dans `configuration/.agents/AGENT_GOVERNANCE.adoc`
